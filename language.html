Language
Detector

Classifying text as English, Italian, or Portuguese using word length statistics — no machine learning, pure statistical distance.

Back to overview

Purpose & Context

This project was built as part of the Programming Design Principles module. The task was to analyse texts from three educational theorists — Dewey (English), Montessori (Italian), and Freire (Portuguese) — and discover whether word length patterns alone could identify a text’s language.

The insight driving the project is that different languages have different average word lengths. By measuring how closely a sample text’s word length distribution matches each reference language, it is possible to classify the sample without understanding any of the words themselves.

Conceptual Explanation

The classifier works in a pipeline of five steps:

  1. Extract word lengths — split the text and measure the length of each word
  2. Build a distribution — count how many words have each length (1, 2, 3…)
  3. Convert to percentages — normalise so texts of different lengths compare fairly
  4. Measure distance — calculate how different a sample’s distribution is from each reference
  5. Classify — return the language with the smallest distance

Why percentages matter

The most important design decision was normalising raw counts to percentages before comparing. Without this, a longer text would always appear more different from a shorter one due to volume alone — not because of language. Percentages remove the size effect.

Step 1 — Extract word lengths

def get_word_lengths(text):
    """
    Extract the length of each word in a text.

    Args:
        text: String containing the text to analyze

    Returns:
        List of integers representing word lengths
    """
    # Your code here
    words = text.split(" ")
    lengths = []
    for word in words:
        lengths.append(len(word))
    return lengths

Step 2 — Build distribution dictionary

def word_length_distribution(text):
    """
    Count how many words have each length.

    Args:
        text: String containing the text to analyze

    Returns:
        Dictionary mapping length to count, e.g. {1: 3, 2: 5, 3: 8, ...}
    """
    # Your code here
    lengths = get_word_lengths(text)
    dist = {}

    for length in lengths:
        if length not in dist:
            dist [length] = 1
        else:
            dist [length] += 1
    return dist

Step 3 — Compare statistics (basic method)

def compare_statistics(text1, text2):
    """
    Calculate distance between two texts using average and median.

    Returns:
        Float representing distance (smaller = more similar)
    """
    # Your code here
   # 1. get the means,do the difference

    mean_of_text1 = sum(get_word_lengths(text1)) / len(get_word_lengths(text1))
    mean_of_text2 = sum(get_word_lengths(text2)) / len(get_word_lengths(text2))

    difference_in_averages = mean_of_text1 - mean_of_text2

    # 2. get the median, do the difference
    # sort data with the list of length, get the index
    sorted_text1 = sorted(get_word_lengths(text1))
    sorted_text2 = sorted(get_word_lengths(text2))

    median1 = sorted_text1[len(sorted_text1)//2]
    median2 = sorted_text2[len(sorted_text2)//2]

    difference_in_median = median1 - median2

    # 3. Add the absolute values of these differences
    absolute_values = abs(difference_in_median) + abs(difference_in_averages)

    # Return total distance
    return absolute_values

Step 4 — Mean Squared Error (the better method)

def mean_squared_error(text1, text2):
    """
    Calculate MSE between percentage distributions of two texts.

    Returns:
        Float representing MSE (smaller = more similar)
    """
    # Your code here
    #1. Dictionary for distribution : What's happening:
    '''
    Call the percentage_distribution() function on text1
    Do the same for text2
    Now we have two dictionaries to compare
    '''

    data_length1 = percentage_distribution(text1)
    data_length2 = percentage_distribution(text2)

   # initial count of sum of squared differences
    total = 0 # Will store the sum of all squared differences
    count = 0 # will track how many word lengths we compare


    # loop through each word length present in text 1
    for length in data_length1:
        percentage_1 = data_length1[length] # Get the percentage for length 1 from text1
        percentage_2 = data_length2.get(length , 0)
        difference_between_percentages = percentage_1 - percentage_2
        diff_squared = difference_between_percentages ** 2
        total += diff_squared
        count += 1
    # Looping through lengths in text 2 that arent in text 1
    for length in data_length2:
        if length not in data_length1:
            percentage_1 = 0 # this is a bit confusing but idea is : if length is new and not processed in the first loop process it now
            percentage_2 = data_length2[length]
            difference_between_percentages = percentage_1 - percentage_2
            diff_squared = difference_between_percentages ** 2
            total += diff_squared
            count += 1
    mse = total / count

    return mse

Step 5 — Predict language

def predict_language(sample_text, english_ref, italian_ref, portuguese_ref):
    """
    Predict the language of a sample text.

    Args:
        sample_text: Text to classify
        english_ref: Reference text in English
        italian_ref: Reference text in Italian
        portuguese_ref: Reference text in Portuguese

    Returns:
        String: "English", "Italian", or "Portuguese"
    """
    # Your code here
    '''
    what are we doing here we're associating the language to its corresponding mse
    '''

    mse_to_english = mean_squared_error(sample_text, english_ref)
    mse_to_italian = mean_squared_error(sample_text, italian_ref)
    mse_to_portuguese = mean_squared_error(sample_text, portuguese_ref)

    # next I will create an if statement that conditionalises what text to classify
    # as what based on distance using the mean squared error

    if mse_to_english < mse_to_italian and mse_to_english < mse_to_portuguese:
        predicted_language = 'English'
    elif mse_to_italian < mse_to_english and mse_to_italian < mse_to_portuguese: # reading this back i could've shortened this but logic is clearer this way
        predicted_language = 'Italian'
    else:
        predicted_language = 'Portuguese'

    return predicted_language

Reflection

One thing that surprised us was that word length alone could be used to identify a language at all. We expected languages to feel very different but did not expect that difference to show up so clearly in something as simple as how long the words are. The most important design decision we made was converting the raw word counts into percentages before comparing texts — without this a longer text could always appear more different from a shorter one just because of its size, not language.

The hardest pair to distinguish was English and Portuguese, with a compare_statistics distance of only 0.44. The easiest was Italian and Portuguese at 5.70 — counterintuitive since they are both Romance languages.

The main limitation is that the predictor requires at least 500 characters to classify reliably and only works for the three languages it was trained on. It may also have learned each specific author’s writing style rather than the language itself, since only one text per language was used as a reference. Adding more reference texts per language would reduce this risk.

def percentage_distribution(text):
    dist = word_length_distribution(text)
    total = sum(dist.values())
    return {length: (count / total) * 100 for length, count in dist.items()}
