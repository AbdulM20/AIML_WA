Purpose & Context

This was the first skills demo of the course and the starting point of everything that followed. The goal was to implement the most fundamental unit in any neural network — a single artificial neuron — without any libraries. Writing it from scratch made the maths concrete before building anything more complex.

A biological neuron fires when it receives enough input. An artificial neuron does the same thing mathematically: it multiplies its input by a weight, adds a bias, and then decides whether to “fire” using an activation function.

Conceptual Explanation

  1. Weighted sum — the input is multiplied by a weight. A larger weight means more influence. Formula: weighted_sum = input × weight + bias
  2. Bias — a constant added to the weighted sum. Lets the neuron fire even when input is zero.
  3. ReLU — if the weighted sum is positive it passes through unchanged; if negative it is clamped to zero.
ReLU activation behaviour
Weighted Sum ReLU Output Why
Positive (e.g. 8)8Passes through unchanged
Zero0Boundary — neuron does not fire
Negative (e.g. −5)0Clamped to zero

Reflection

Building this made the purpose of ReLU much clearer than reading about it. A negative weighted sum means the inputs and weights did not combine strongly enough — ReLU turns that into a clean zero rather than letting a small negative value propagate and destabilise a larger network.

Two bugs appeared during development. The first was using operators instead of commas in the function call. The second was a missing underscore in the function name in a print call, which Python read as a syntax error. Both were caught immediately but they reinforced how precise Python’s syntax needs to be.

This neuron only handles a single input and a single weight. Real neurons take many inputs simultaneously, each with its own weight — that extension came in the neural network projects that followed.
