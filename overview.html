Project Summaries

What Each Project Covers

01

Artificial Neuron

The starting point of the course. Three functions building from the ground up: a ReLU activation, a weighted sum, and an activated neuron combining both. Demonstrates why activation functions are necessary and how a neuron “fires.”

  • Python
  • ReLU
  • Weighted Sum
02

Language Detector

A statistical classifier that identifies language from word length distributions alone. Implements two distance methods — basic statistics and Mean Squared Error — and compares their effectiveness. Requires a minimum of 500 characters for reliable results.

  • Python
  • Statistics
  • MSE
  • Classification
03

Neural Network — Single Class

A complete neural network in a single class. Forward pass, cross-entropy loss, and backpropagation all written from first principles. Three architectures compared on MNIST: tiny (6,370 params), medium (52,650), and large (535,818). The large network overfits; medium offers the best real-world balance.

  • Python
  • Backpropagation
  • MNIST
  • Softmax
04

Neural Network — Multiple Classes

A full OOP redesign of the neural network. Separate classes for layers, activations, and output functions, all connected through a shared forward() interface. Two architectures (64 and 128 hidden nodes) trained on 2,000 MNIST examples, reaching 89.7% and 90.7% test accuracy respectively.

  • Python
  • OOP
  • Inheritance
  • MNIST

Looking Back

Overall Reflection

These four projects represent a clear progression from first principles to applied machine learning. The artificial neuron was the smallest possible starting point — a single mathematical operation — but every concept that followed grew from it directly. ReLU, weighted sums, and the idea that a network decides whether to “fire” all come back to that first function.

The language detector was a deliberate detour away from neural networks. It showed that machine learning does not always mean deep learning — a simple statistical distance measure built from scratch was enough to classify three languages with high accuracy, given sufficient sample size. The design decisions made there (normalising to percentages, choosing MSE over a basic distance) mirror the kinds of decisions made when building larger models.

The two neural network projects showed what changes when you move from a working implementation to a well-designed one. The single-class network worked. The modular OOP version was easier to experiment with, easier to extend, and far easier to reason about. That design shift was the most important thing I took away from the final skills demo.

The next direction I want to pursue is testing deeper architectures (more hidden layers, not just wider ones) and applying these networks to a domain I care about — financial time series data.
